## 📚 기초 개념: 입력(Input)과 출력(Output)의 차이

### **입력(Input) 토큰이란?**
```
사용자가 보내는 메시지, 프롬프트, 시스템 지시사항의 길이

예시: "한국 GDP를 분석하고 경제 전망을 제시하세요"
      ↓
      이 문장을 토큰화 = ~20개 입력 토큰
```

**특징:**
- 모델이 "읽는" 데이터
- 더 저렴하게 책정됨 (모델이 이미 처리했으니)
- 길이 제한: 컨텍스트 윈도우

### **출력(Output) 토큰이란?**
```
모델이 생성하여 사용자에게 보내는 답변의 길이

예시: "2024년 한국의 GDP는 약 1.4조 달러로...
      (100개 단어 = ~150개 출력 토큰)"
```

**특징:**
- 모델이 "만드는" 데이터
- 더 비싸게 책정됨 (생성 비용이 큼)
- 생성할수록 비용 증가

### **가격 차이 예시**
```
DeepSeek V4:
입력  $0.30 (저렴)  ← 읽기만 함
출력  $0.50 (비쌈)  ← 생성해야 함

Minimax M2.7:
입력  $0.30 (동일)
출력  $1.20 (훨씬 비쌈)  ← 생성이 더 복잡

💡 같은 입력에 대해 긴 답변을 원하면 Minimax가 더 비쌈
```
